# UI仕様書 - メイン画面（ダッシュボード）

> **スコープ**: メイン画面（ダッシュボード）のレイアウト、コンポーネント、インタラクションの UI 仕様を扱う。バックエンドのポーリング設計は [architecture.md §6](../architecture.md)、API 仕様は [openapi.yaml](../openapi.yaml) を参照。

## 1. 概要

| 項目 | 内容 |
|---|---|
| 画面名 | メイン画面（ダッシュボード） |
| パス | `/` |
| 概要 | 左サイドバーにカテゴリ一覧、右メインエリアに選択カテゴリのコンテンツ一覧を表示する |
| 認証 | 要ログイン（未ログイン時は `/login` にリダイレクト） |

---

## 2. レイアウト構成

```
+------------------------------------------------------------------+
| ヘッダー                                                           |
+-------------------+----------------------------------------------+
|                   |  カテゴリ名  [今すぐポーリング]  ソート  フィルター  |
|  サイドバー        |  ─────────────────────────────────────────── |
|  （カテゴリ一覧）   |  コンテンツリスト（無限スクロール）              |
|                   |                                              |
|                   |                                              |
+-------------------+----------------------------------------------+
```

- **PC（768px以上）**: サイドバー固定表示（左）＋メインエリア（右）の2カラム構成
- **スマートフォン（768px未満）**: サイドバーはハンバーガーメニューやドロワーで表示。デフォルト非表示

## 3. ヘッダー

### 表示内容

```
+------------------------------------------------------------------+
| MySubChs  |  チャンネル管理  カテゴリ管理  設定  |  [ユーザーアイコン ▼] |
+------------------------------------------------------------------+
```

| 要素 | 説明 |
|---|---|
| アプリ名「MySubChs」 | ロゴ兼ホームリンク（`/` へ遷移） |
| グローバルナビ | 「チャンネル管理」`/channels`、「カテゴリ管理」`/categories`、「設定」`/settings` へのリンク |
| ユーザーメニュー | Googleアカウントのアイコン・名前を表示。クリックでドロップダウンを展開し「ログアウト」を表示 |

- スマートフォンではグローバルナビをハンバーガーメニュー（またはボトムナビゲーション）に格納することも可

---

## 4. サイドバー（カテゴリ一覧）

### レイアウト

```
+-------------------+
| カテゴリ1          |
| カテゴリ2          |
| カテゴリ3          |
| ─────────────── |
| 未分類             |
+-------------------+
```

### 表示ルール

- カテゴリは `sortOrder` 昇順で一覧表示する
- 「未分類」はリストの**最下部に固定**表示する特別カテゴリ（ユーザー作成カテゴリとは区切り線で分割）
- 選択中のカテゴリはハイライト表示する（背景色の変化など）
- 初期状態（画面初回表示）では、カテゴリ一覧の先頭（`sortOrder` が最小）のカテゴリが選択される
  - カテゴリが1件も存在しない場合は「未分類」が選択される

### カテゴリ行の表示内容

| 要素 | 説明 |
|---|---|
| カテゴリ名 | 最大1行・テキスト省略（overflow ellipsis） |

- サイドバー上での編集・削除操作は行わない（カテゴリ管理画面 `/categories` で行う）

### 空の状態

- カテゴリが1件もない場合でも「未分類」は常に表示するため、特別な空状態メッセージは不要

## 5. メインエリア

### 5.1 エリアヘッダー

```
+----------------------------------------------+
| カテゴリ名  [今すぐポーリング]  [新しい順 ▼]    |
| [後で見るのみ表示]  [キャンセル済みも表示]        |
+----------------------------------------------+
```

| 要素 | 説明 |
|---|---|
| カテゴリ名 | 選択中のカテゴリ名（「未分類」の場合は「未分類」と表示） |
| 「今すぐポーリング」ボタン | 選択中カテゴリの手動ポーリングを実行する（§5.2 参照）。「未分類」選択中は非活性（クリック不可）。ツールチップで「未分類はポーリング対象外です」と表示する |
| ソートトグルボタン | `新しい順`（降順） ↔ `古い順`（昇順）を切り替える。切り替え時はカーソルをリセットして先頭から再取得 |
| 後で見るフィルター | トグルボタン。ONにすると「後で見る」がONのコンテンツのみ表示（`watchLaterOnly=true`）。OFFがデフォルト |
| キャンセル済みフィルター | トグルボタン。ONにすると `status=CANCELLED` のコンテンツも表示（`includeCancelled=true`）。OFFがデフォルト |

- ソート状態・フィルター状態はブラウザのURLクエリパラメータで管理し、リロード後も保持する

### 5.2 「今すぐポーリング」ボタンの挙動

| 状態 | 表示 |
|---|---|
| 通常 | 「今すぐポーリング」（活性） |
| 実行中（非同期） | ボタンを無効化し、スピナーを表示 |
| クールダウン中 | ボタンを非活性化し「あと X分 X秒」を表示 |
| クールダウン終了後 | 通常状態に戻る |
| 「未分類」選択中 | 常に非活性。ツールチップ「未分類はポーリング対象外です」を表示 |

- クリック後、`POST /api/categories/{categoryId}/poll` を呼び出す
- ポーリングはバックグラウンドで非同期実行される（APIはジョブのキューへの追加成功のみ応答する）
- APIから `429 Too Many Requests` が返った場合（クールダウン中）：ボタンをクールダウン状態にし、レスポンスの `error.retryAfter` 秒のカウントダウンを表示する

**POST 後のジョブ完了検知:**

1. ボタンを「実行中」状態（スピナー・無効化）に設定
2. `GET /api/categories/{categoryId}/poll/status` を **3秒間隔・最大100回**（上限5分）でポーリング開始

ステータス確認ループの遷移ルール:

| `status` 値 / 条件 | 動作 |
|---|---|
| `"active"` / `"waiting"` | 「実行中」状態を維持、ポーリング継続 |
| `"completed"` / `"none"` | ポーリング停止 → コンテンツ一覧を refetch → `cooldownRemaining > 0` なら「クールダウン中」状態へ |
| `"failed"` | ポーリング停止 → エラートーストを表示 → コンテンツ一覧を refetch → `cooldownRemaining > 0` なら「クールダウン中」状態へ |
| 100回到達（タイムアウト） | ポーリング停止 → タイムアウトエラートーストを表示 → 通常状態に戻す |

- `"none"` は `"completed"` と同じ扱い（ジョブが既にクリーンアップされた状態）。`cooldownRemaining` で状態を判断する
- `"failed"` 時もクールダウンは継続（キュー登録時点でクールダウン開始済みのため）

**ページロード時の状態復元:**

1. `GET /api/categories/{categoryId}/poll/status` を1回呼び出す
2. 結果に応じてボタン初期状態を決定:
   - `status = "active"` or `"waiting"` → 「実行中」状態でポーリング再開（3秒間隔・最大100回）
   - `status = "completed"` or `"none"` かつ `cooldownRemaining > 0` → 「クールダウン中」状態で表示
   - それ以外 → 通常状態

### 5.3 コンテンツリスト

- 無限スクロール方式。1回のリクエストで最大20件取得する（`GET /contents?categoryId=...&cursor=...`）
- リスト下部に到達したら次のページを自動取得する（TanStack Query の `useInfiniteQuery` を使用）
- 取得中はリスト下部にローディングインジケーターを表示する
- `meta.hasNext = false` の場合は「すべて読み込みました」などの終端メッセージを表示する（省略可）

## 6. コンテンツリストアイテム

```
+----------------------------------------------+
| [バッジ]  タイトル（YouTubeへのリンク）  [🔖]  |
|          チャンネル名  ·  日時テキスト          |
+----------------------------------------------+
```

### 表示項目

| 要素 | 説明 |
|---|---|
| ステータスバッジ | `LIVE`（配信中）/ `UPCOMING`（配信予定）/ `ARCHIVED`（アーカイブ）/ `CANCELLED`（キャンセル済み）をバッジ表示。バッジのスタイルは下記参照 |
| タイトル | コンテンツのタイトル。クリックすると YouTube の該当ページを**新しいタブ**で開く（`target="_blank" rel="noopener noreferrer"`）。最大2行・テキスト省略 |
| 後で見るトグル | ブックマークアイコン（🔖）。ONの場合は塗りつぶし・アクティブスタイル、OFFは線のみ。クリックでトグル |
| チャンネル名 | チャンネルのテキスト名（アイコンは表示しない） |
| 日時テキスト | コンテンツの種別・ステータスに応じて表示する日時（下記参照） |

**サムネイルは表示しない**（要件定義 §2.3 より）。

### バッジスタイル

バッジは `type` と `status` の組み合わせで決定する。

| type | status | 表示テキスト | スタイル |
|---|---|---|---|
| LIVE | LIVE | 配信中 | 赤・塗りつぶし（shadcn `destructive`） |
| LIVE | UPCOMING | 配信予定 | 青・塗りつぶし（Tailwind `bg-blue-600 text-white`） |
| LIVE | ARCHIVED | アーカイブ | 緑・塗りつぶし（Tailwind `bg-green-600 text-white`） |
| VIDEO | ARCHIVED | 動画 | 緑・枠のみ（Tailwind `border-green-600 text-green-600`） |
| LIVE | CANCELLED | キャンセル済み | グレー・枠のみ・文字薄め（shadcn `outline` + `text-muted-foreground`） |

- `type=VIDEO` かつ `status=ARCHIVED` 以外の組み合わせは仕様上発生しない（VIDEO は ARCHIVED 固定）

### 日時テキストの表示ルール

| type | status | 表示する日時 | 表示形式例 |
|---|---|---|---|
| VIDEO | ARCHIVED | `publishedAt` | `2026/03/03 15:00` |
| LIVE | UPCOMING | `scheduledStartAt` | `配信予定: 2026/03/03 20:00` |
| LIVE | LIVE | `actualStartAt` | `配信開始: 2026/03/03 19:58` |
| LIVE | ARCHIVED | `actualStartAt`（なければ `scheduledStartAt`） | `2026/03/03 19:58` |
| LIVE | CANCELLED | `scheduledStartAt`（なければ非表示） | `予定: 2026/03/03 20:00` |

- 日時は端末のローカルタイムゾーンで表示する
- `contentAt` はソートに使用するが、表示には使用しない

### 後で見るトグルの挙動

| 操作 | API呼び出し | 結果 |
|---|---|---|
| OFFからONにする | `PUT /watch-later/{contentId}` | フラグをONにする（`addedVia=MANUAL, expiresAt=null`）|
| ONからOFFにする | `DELETE /watch-later/{contentId}` | フラグをOFFにし `removedVia='MANUAL'` をセット |

- API呼び出し中はアイコンをローディング状態にする（操作を連打できないように制御）
- 楽観的更新（Optimistic Update）でレスポンスを待たずにUIを即時更新し、失敗時にはロールバックする

## 7. 各種状態の表示

### 7.1 初回ログイン直後（チャンネル取得中）

チャンネルが0件の間（初回ログイン後、初回ポーリングが完了するまで）：

```
+----------------------------------------------+
|                                              |
|   YouTubeの登録チャンネルを取得中です...         |
|   [ローディングスピナー]                        |
|                                              |
+----------------------------------------------+
```

- ポーリングが完了するまで数十秒〜数分かかる場合がある
- チャンネル数が0より大きくなったら通常の表示に切り替える（TanStack Query のポーリングで監視）

### 7.2 選択カテゴリにコンテンツが0件

```
+----------------------------------------------+
|                                              |
|   このカテゴリにはまだコンテンツがありません。    |
|   次のポーリング後に表示されます。               |
|                                              |
+----------------------------------------------+
```

### 7.3 選択カテゴリにチャンネルが0件（コンテンツリストが空かつチャンネル未割り当て）

- 上記 §7.2 と同じ空状態メッセージを表示する
  - チャンネルがないことを強調したい場合：「このカテゴリにはチャンネルが割り当てられていません。[チャンネル管理画面]で割り当ててください。」も可

### 7.4 「後で見るのみ表示」フィルターで0件

```
「後で見る」に登録されたコンテンツはありません。
```

### 7.5 コンテンツリスト読み込み中（初回ロード）

- メインエリアにスケルトンローダーを表示する（リストアイテムの骨格を3〜5件分）

### 7.6 APIエラー時

- エラーが発生した場合はトースト通知（または画面上部のバナー）でエラーメッセージを表示する
- 「後で見る」のトグル失敗時はトーストでエラーを通知し、楽観的更新をロールバックする
- **401（セッション失効）** の場合は [architecture.md §8](../architecture.md) のグローバルハンドラーに従い、トースト表示後に `/login` へリダイレクトする

## 8. データ取得・更新フロー

### カテゴリ一覧

- `GET /categories` でサイドバーに表示するカテゴリ一覧を取得する
- TanStack Query でキャッシュし、画面のフォーカス復帰時に再取得する

### コンテンツ一覧

- カテゴリ選択時に `GET /contents?categoryId={id}&order=desc&limit=20` を呼び出す
- 「未分類」選択時は `categoryId=uncategorized` を指定する
- TanStack Query の `useInfiniteQuery` で無限スクロールを実装する
- ソートトグル・フィルター変更時はクエリをリセットして先頭から再取得する

## 9. レスポンシブ対応

| ブレークポイント | レイアウト |
|---|---|
| PC（768px以上） | サイドバー固定（左、幅 200〜240px 程度）＋メインエリア |
| スマートフォン（768px未満） | サイドバーを非表示にし、上部にカテゴリ選択ドロップダウンまたはドロワーで代替 |

- コンテンツリストアイテムはモバイルでも1カラム表示（グリッドレイアウトは使用しない）
- 後で見るトグルはモバイルでもタップしやすいサイズ（最低44×44px）を確保する

## 10. 関連API

| メソッド | パス | 用途 |
|---|---|---|
| GET | `/api/categories` | サイドバーのカテゴリ一覧取得 |
| GET | `/api/contents` | コンテンツ一覧取得（無限スクロール） |
| POST | `/api/categories/{categoryId}/poll` | 手動ポーリング実行 |
| GET | `/api/categories/{categoryId}/poll/status` | ポーリングジョブステータス確認 |
| PUT | `/api/watch-later/{contentId}` | 「後で見る」をONにする |
| DELETE | `/api/watch-later/{contentId}` | 「後で見る」をOFFにする |
