# UI仕様書 - ログイン画面

> **スコープ**: ログイン画面のレイアウトとインタラクションの UI 仕様を扱う。認証フローの技術詳細は [architecture.md §5](../architecture.md) を参照。

## 1. 概要

| 項目 | 内容 |
|---|---|
| 画面名 | ログイン画面 |
| パス | `/login` |
| 概要 | Googleアカウントでログインするための画面。「Googleでログイン」ボタンのみを表示するシンプルな構成 |
| 認証 | 不要（未ログイン時のエントリーポイント）。ログイン済みの場合は `/` にリダイレクト |

---

## 2. レイアウト構成

```
+------------------------------------------------------------------+
|                                                                  |
|                                                                  |
|                        MySubChs                                  |
|               YouTubeサブスクリプション管理                         |
|                                                                  |
|                  [G Googleでログイン]                             |
|                                                                  |
|       （エラー時のみ）                                             |
|       ⚠ ログインに失敗しました。もう一度お試しください。              |
|                                                                  |
|                                                                  |
+------------------------------------------------------------------+
```

- 画面全体を縦横中央揃えで表示する（ビューポート全体を使用）
- PC・スマートフォンともに同一レイアウト

---

## 3. 表示要素

| 要素 | 説明 |
|---|---|
| アプリ名「MySubChs」 | 見出しテキスト（ロゴとして表示） |
| サブテキスト | 「YouTubeサブスクリプション管理」（アプリ説明の補足） |
| 「Googleでログイン」ボタン | クリックでGoogleのOAuth認証フローを開始する |
| エラーメッセージ | OAuth失敗時のみ表示（下記参照） |

---

## 4. ボタンの挙動

- クリックすると NextAuth の `signIn('google', { callbackUrl: '/' })` を呼び出す
  - `callbackUrl` は明示的に `/` を指定する（NextAuth のデフォルト動作に依存しない）
- 認証成功後は `/` にリダイレクトする
- クリック後、認証結果が返るまでボタンをローディング状態（`disabled` ＋スピナー）にする

---

## 5. エラー表示

NextAuthがコールバックURL（`/api/auth/callback/google`）のエラーを `/login?error=<code>` のクエリパラメータとして返す場合、画面にエラーメッセージを表示する。

| `error` パラメータ | 表示メッセージ |
|---|---|
| `AccessDenied` | 「アクセスが拒否されました。Googleアカウントへのアクセスを許可してください。」 |
| その他（`OAuthCallback` など） | 「ログインに失敗しました。もう一度お試しください。」 |
| パラメータなし | エラーメッセージを表示しない |

- エラーメッセージはボタンの下に表示する
- エラーがある場合でもボタンは活性状態を維持し、再試行できる

---

## 6. ログイン済みの場合のリダイレクト

- セッションが有効な状態で `/login` にアクセスした場合、`/` にリダイレクトする
- `middleware.ts` で処理する（Next.js App Router の推奨方式。ページコンポーネント側でのセッションチェックは不要）

---

## 7. レスポンシブ対応

| ブレークポイント | 差異 |
|---|---|
| PC（768px以上） | 中央揃えブロックの最大幅を 360px 程度に制限する。ボタン幅はブロック幅に合わせる（`w-full`） |
| スマートフォン（768px未満） | ボタンを全幅（`w-full`）にする |
