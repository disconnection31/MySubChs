# UI仕様書 - カテゴリ管理画面

> **スコープ**: カテゴリ管理画面のレイアウト、コンポーネント、インタラクションの UI 仕様を扱う。カテゴリ別設定（通知・ポーリング・後で見る）のデフォルト値は [database.md](../database.md) を参照。

## 1. 概要

| 項目 | 内容 |
|---|---|
| 画面名 | カテゴリ管理画面 |
| パス | `/categories` |
| 概要 | カテゴリのCRUD・表示順の並び替え・カテゴリ別設定（ポーリング・通知・後で見る）を管理する |
| 認証 | 要ログイン（未ログイン時は `/login` にリダイレクト） |

---

|     ▼ 設定展開中                                                   |
|     ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ |

|     通知:       新着動画通知  ─────────────────── [ON ●]         |
|                 ライブ開始通知 ─────────────────── [ON ●]         |

- **PC（768px以上）**: 1カラムのカードリスト。各行にドラッグハンドル・カテゴリ名・操作ボタン・設定アコーディオン

## 3. クォータ警告

ページヘッダーとカテゴリリストの間に表示する。

### 表示内容

| 要素 | 説明 |
|---|---|
| 推定1日クォータ使用量 | `estimatedDailyQuota / quotaDailyLimit` の形式で常に表示する（例：「推定1日クォータ使用量: 4,900 / 10,000 units」） |
| 警告メッセージ | `estimatedDailyQuota > quotaWarningThreshold` の場合のみ表示する |

- 警告メッセージ例：「全カテゴリの合計クォータが警告しきい値を超えています。一部のカテゴリのポーリング間隔を長くすることを推奨します。」
- `estimatedDailyQuota`・`quotaWarningThreshold`・`quotaDailyLimit` は `GET /api/settings` のレスポンス値を使用する（ハードコード禁止）

### カテゴリ別ポーリング間隔変更後のクォータ表示更新

カテゴリのポーリング間隔を変更した場合、PATCH 成功後に `GET /api/settings` を再フェッチし、サーバーが再計算した `estimatedDailyQuota` で警告表示を更新する。フロントエンドはクォータの再計算を行わない。

## 6. カテゴリ追加

「+ カテゴリを追加」ボタンをクリックすると、リストの**末尾**に新規入力行が展開される。

```
+------------------------------------------------------------------+
| [≡]  [カテゴリ名を入力...]          [追加]  [キャンセル]           |
+------------------------------------------------------------------+
```

| 要素 | 説明 |
|---|---|
| テキスト入力 | プレースホルダー「カテゴリ名を入力…」。フォーカスが自動で当たる |
| 「追加」ボタン | 入力値が空の場合は非活性。クリックで `POST /api/categories` を呼び出す |
| 「キャンセル」ボタン | 入力行を閉じる |

- `Enter` キーで「追加」と同等の動作をする
- `Escape` キーで「キャンセル」と同等の動作をする
- 追加成功後、入力行を閉じてリストを再取得する。追加されたカテゴリが末尾に表示される
- API が `409 Conflict`（`CATEGORY_NAME_DUPLICATE`）を返した場合、入力フィールドの下にインラインエラー「同じ名前のカテゴリがすでに存在します」を表示し、入力フィールドを維持する

## 7. カテゴリリスト

### 7.1 表示ルール

- `sortOrder` 昇順で全カテゴリを表示する
- カテゴリが0件の場合は §11.1 の空状態を表示する

### 7.2 カテゴリ行（通常状態）

```
+------------------------------------------------------------------+
| [≡]  カテゴリ名              [🗑️]  [設定 ▼]   |
+------------------------------------------------------------------+
```

| 要素 | 説明 |
|---|---|
| ドラッグハンドル `[≡]` | ドラッグ&ドロップで並び替えるためのハンドル。PCのみ表示 |
| カテゴリ名 | テキスト表示。クリックするとインライン編集モードに切り替わる（§8 参照） |
| 削除ボタン `[🗑️]` | クリックで確認ダイアログを表示する（§9 参照） |
| 「設定 ▼」ボタン | クリックで設定アコーディオンを展開・折りたたむ（§7.3 参照） |

### 7.3 設定アコーディオン（展開時）

「設定 ▼」ボタンをクリックすると、その行の下に設定パネルが展開される。複数カテゴリを同時に展開できる。

```
+------------------------------------------------------------------+
| ▼ ポーリング                                                      |
|   定期ポーリング ──────────────────────────────── [ON ●]        |
|   ポーリング間隔 ─────────────── [グローバル設定を使用（30分） ▼] |
|   ← 定期ポーリングがOFFの場合は非活性                              |
|                                                                  |
| ▼ 通知                                                           |
|   新着動画通知 ────────────────────────────────── [ON ●]        |
|   ライブ開始通知 ──────────────────────────────── [ON ●]        |
|   配信予定の通知 ──────────────────────────────── [OFF ○]       |
|                                                                  |
| ▼ 後で見る                                                        |
|   新着コンテンツに自動フラグを付ける ───────────── [OFF ○]         |
|   自動失効時間  [  失効なし  ▼ ]  ← 上記ONの場合のみ表示          |
+------------------------------------------------------------------+
```

#### ポーリング設定

| 設定項目 | 説明 | デフォルト |
|---|---|---|
| 定期ポーリング | OFFにするとこのカテゴリのチャンネルが定期ポーリング対象外になる。OFFでもダッシュボードの「今すぐポーリング」ボタンから手動ポーリングは実行可能 | ON |
| ポーリング間隔 | 定期ポーリングがOFFの場合は非活性にする。グローバル設定と異なる間隔をこのカテゴリに設定できる | グローバル設定を使用 |

**ポーリング間隔セレクトの選択肢:**

| 表示テキスト | 値 |
|---|---|
| グローバル設定を使用（現在: {グローバル値}分） | null |
| 5分 | 5 |
| 10分 | 10 |
| 30分 | 30 |
| 1時間 | 60 |

- 「グローバル設定を使用」の表示テキスト内のカッコには `GET /api/settings` で取得した現在のグローバル値を表示する
- 変更時は即座に `PATCH /api/categories/{categoryId}/settings` を呼び出す（`pollingIntervalMinutes` フィールドを送信）

#### 通知設定

| 設定項目 | 説明 | デフォルト |
|---|---|---|
| 新着動画通知 | 新しい動画が投稿されたときにWeb Push通知を送る | ON |
| ライブ開始通知 | ライブ配信が開始したときにWeb Push通知を送る | ON |
| 配信予定の通知 | 新しいライブ予約が登録されたときにWeb Push通知を送る | OFF |

#### 後で見る設定

| 設定項目 | 説明 | デフォルト |
|---|---|---|
| 新着コンテンツに自動フラグを付ける | ONにすると、このカテゴリの新着コンテンツに自動で「後で見る」フラグが付く | OFF |
| 自動失効時間 | 「自動フラグを付ける」がONのときのみ表示する。選択肢：1日 / 3日 / 1週間 / 2週間 / 失効なし | 失効なし |

#### 設定変更の挙動

- 各トグル・セレクトの変更時に**即座に** `PATCH /api/categories/{categoryId}/settings` を呼び出す（確認ダイアログなし）
- API呼び出し中はトグルをローディング状態にする（操作を連打できないように制御）
- 失敗時はトースト通知でエラーを表示し、変更前の値に戻す（楽観的更新のロールバック）

## 8. カテゴリ名インライン編集

カテゴリ名のテキスト部分をクリックすると、その場でテキスト入力フィールドに切り替わる。

```
通常:   [≡]  カテゴリ名              [🗑️]  [設定 ▼]
編集中: [≡]  [カテゴリ名|]  [保存]  [🗑️]  [設定 ▼]
```

| 操作 | 動作 |
|---|---|
| テキストをクリック | 入力フィールドに切り替わり、現在の名前が入力済みの状態でフォーカスされる |
| `Enter` / 「保存」クリック | `PATCH /api/categories/{categoryId}` を呼び出す。成功後、テキスト表示に戻る |
| `Escape` / フォーカスアウト | 変更を破棄してテキスト表示に戻る |

- 入力値が空の場合、「保存」ボタンは非活性。`Enter` 操作も無効
- 入力値が変更されていない（元の名前と同一）場合は保存APIを呼び出さずにテキスト表示に戻る
- API が `409 Conflict`（`CATEGORY_NAME_DUPLICATE`）を返した場合、入力フィールドの下にインラインエラー「同じ名前のカテゴリがすでに存在します」を表示し、入力フィールドを維持する
- それ以外の保存失敗時はトースト通知でエラーを表示し、入力フィールドを維持する

---

## 9. カテゴリ削除

削除ボタン（🗑️）をクリックすると確認ダイアログを表示する。

```
+---------------------------------------+
|  カテゴリを削除                          |
|  ─────────────────────────────────── |
|  カテゴリ「{カテゴリ名}」を削除します。   |
|  所属するチャンネルは未分類になります。   |
|                                       |
|             [キャンセル]  [削除する]     |
+---------------------------------------+
```

- 「削除する」ボタンは強調色（赤系）で表示する
- 「削除する」クリックで `DELETE /api/categories/{categoryId}` を呼び出す
- 削除成功後、ダイアログを閉じてリストを再取得する
- 削除失敗時はダイアログ内またはトーストでエラーメッセージを表示する

---

## 10. 並び替え（ドラッグ&ドロップ）

- PCのみ対応。スマートフォンでは行の先頭のドラッグハンドルを非表示にする
- ドラッグハンドル（`[≡]`）をドラッグしてカテゴリの表示順を変更できる
- ドロップ完了後（ドラッグエンド時）に `PATCH /api/categories/reorder` を呼び出す
  - リクエストボディ: `{ orderedIds: ["id1", "id2", ...] }` （新しい表示順のID配列）
- API呼び出し中はリスト全体を非活性化し、操作を受け付けない
- 並び替え失敗時はトースト通知でエラーを表示し、元の順序に戻す（楽観的更新のロールバック）

## 11. 各種状態の表示

### 11.1 カテゴリが0件

```
+------------------------------------------------------------------+
|                                                                  |
|   カテゴリがまだありません。                                         |
|   「+ カテゴリを追加」からカテゴリを作成してください。                |
|                                                                  |
+------------------------------------------------------------------+
```

### 11.2 カテゴリ一覧読み込み中

- カテゴリ行のスケルトンローダーを3件分表示する

### 11.3 APIエラー時

- カテゴリ一覧の取得失敗：画面にエラーメッセージを表示する（「カテゴリの取得に失敗しました。再読み込みしてください。」）
- 設定変更・名前編集・削除の失敗：トースト通知でエラーメッセージを表示する
- **401（セッション失効）** の場合は [architecture.md §8](../architecture.md) のグローバルハンドラーに従い、トースト表示後に `/login` へリダイレクトする

## 13. データ取得・更新フロー

| 操作 | APIコール |
|---|---|
| 初回表示 | `GET /api/categories`、`GET /api/settings`（グローバルポーリング間隔・クォータ情報の取得） |
| カテゴリ追加 | `POST /api/categories` → リスト再取得 |
| カテゴリ名編集 | `PATCH /api/categories/{categoryId}` → リスト再取得 |
| カテゴリ削除 | `DELETE /api/categories/{categoryId}` → リスト再取得 |
| 並び替え | `PATCH /api/categories/reorder` → リスト再取得 |
| 設定変更（ポーリング間隔以外） | `PATCH /api/categories/{categoryId}/settings`（楽観的更新） |
| ポーリング間隔変更 | `PATCH /api/categories/{categoryId}/settings` → `GET /api/settings` 再フェッチ |

- カテゴリ一覧は TanStack Query でキャッシュし、画面のフォーカス復帰時に再取得する
- `GET /api/settings` はポーリング間隔セレクトの「グローバル設定を使用（現在: N分）」表示とクォータ警告に必要。TanStack Query でキャッシュし、設定画面と同一クエリキーを使用することで既にキャッシュ済みの場合は再フェッチしない
- ポーリング間隔変更時のみ、PATCH 成功後に `GET /api/settings` を再フェッチしてクォータ表示を更新する
- ポーリング間隔以外の設定変更は楽観的更新を適用し、失敗時にロールバックする
