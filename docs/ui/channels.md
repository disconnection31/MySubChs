# UI仕様書 - チャンネル管理画面

> **スコープ**: チャンネル管理画面のレイアウト、コンポーネント、インタラクションの UI 仕様を扱う。チャンネル同期フローの詳細は [architecture.md §5](../architecture.md) を参照。

## 1. 概要

| 項目 | 内容 |
|---|---|
| 画面名 | チャンネル管理画面 |
| パス | `/channels` |
| 概要 | YouTube登録チャンネルの一覧表示、カテゴリへの割り当て変更、登録解除（ソフトデリート）を管理する |
| 認証 | 要ログイン（未ログイン時は `/login` にリダイレクト） |

## 5. フィルター

```
表示: [アクティブのみ ▼]
```

| 選択肢 | APIパラメータ | 説明 |
|---|---|---|
| アクティブのみ（デフォルト） | `isActive=true` | 有効なチャンネルのみ表示 |
| 解除済みのみ | `isActive=false` | 解除済みチャンネルのみ表示 |

- フィルター変更時はチャンネルリストを再取得する
- フィルター状態はURLクエリパラメータ（`?filter=active` / `?filter=inactive`）で管理し、リロード後も保持する

## 6. チャンネルリスト

### 6.1 グルーピング・表示順

- チャンネルは**カテゴリ別にグルーピング**して表示する
- グループの表示順：カテゴリの `sortOrder` 昇順。**「未分類」グループは末尾に固定**
- 各グループ内のチャンネルはチャンネル名の昇順（辞書順）で表示する
- 現在のフィルター条件に一致するチャンネルが存在しないカテゴリグループは表示しない
- グルーピングのためにカテゴリ一覧（`GET /api/categories`）を別途取得し、クライアントで結合する

### 6.2 グループヘッダー

```
▼ ゲーム実況 (3)
```

| 要素 | 説明 |
|---|---|
| カテゴリ名 | カテゴリの名前（未分類の場合は「未分類」） |
| チャンネル数 | そのグループ内の現在のフィルター条件に合致するチャンネル数 |

- グループヘッダーをクリックすると折りたたみ/展開を切り替えられる（デフォルトは全展開）
- 折りたたみ状態は展開時に ▼、折りたたみ時に ▶ で視覚的に示す
- 折りたたみ状態はページリロード時にリセットされる（永続化しない）

### 6.3 チャンネル行（アクティブ）

```
+------------------------------------------------------------------+
| [icon]  チャンネル名              [カテゴリ ▼]  [解除]             |
+------------------------------------------------------------------+
```

| 要素 | 説明 |
|---|---|
| チャンネルアイコン | `iconUrl` を 32×32px のアイコンで表示。`iconUrl` が null の場合はプレースホルダー画像（グレーの人物アイコン等）を表示 |
| チャンネル名 | テキスト表示。最大1行・テキスト省略（overflow ellipsis） |
| カテゴリセレクトボックス | 現在の所属カテゴリを選択した状態で表示。変更すると即時API更新（§6.5 参照） |
| 解除ボタン | クリックで確認ダイアログを表示（§7 参照）。アクティブチャンネルのみ表示 |

### 6.4 チャンネル行（解除済み）

```
+------------------------------------------------------------------+
| [icon]  チャンネル名              [解除済み]                        |
+------------------------------------------------------------------+
```

- カテゴリセレクトボックスと解除ボタンは表示しない
- 「解除済み」バッジ（グレー・枠のみ、`outline` スタイル）を表示する
- 解除済みチャンネルの復元は設定画面の手動再同期（`POST /api/settings/sync-channels`）でのみ行う

### 6.5 カテゴリセレクトボックスの挙動

**選択肢**：「未分類」＋ユーザー作成カテゴリ（`sortOrder` 昇順）

| 操作 | APIコール | リクエスト |
|---|---|---|
| 別のカテゴリを選択 | `PATCH /api/channels/{channelId}` | `{ categoryId: "<選択したカテゴリのID>" }` |
| 「未分類」を選択 | `PATCH /api/channels/{channelId}` | `{ categoryId: null }` |

- 選択変更時に即座にAPIを呼び出す（「保存」ボタンは不要）
- 楽観的更新でセレクトボックスの値を即時反映し、成功後にリスト全体を再取得する（チャンネルが対応するグループに移動する）
- API呼び出し中はセレクトボックスを `disabled` 状態にする
- 失敗時はトースト通知でエラーを表示し、選択値を元のカテゴリに戻す（ロールバック）

## 7. チャンネル登録解除

解除ボタンをクリックすると確認ダイアログを表示する。

```
+---------------------------------------------+
|  チャンネルを解除                              |
|  ─────────────────────────────────────────  |
|  「{チャンネル名}」をアプリから解除します。       |
|                                             |
|  ※ YouTubeの登録状態は変更されません。          |
|  ※ 再度表示するには設定画面から再同期してください。 |
|                                             |
|                [キャンセル]  [解除する]         |
+---------------------------------------------+
```

- 「解除する」ボタンは強調色（赤系）で表示する
- 「解除する」クリックで `PATCH /api/channels/{channelId}` (`{ isActive: false }`) を呼び出す
- 解除成功後、ダイアログを閉じてリストを再取得する（アクティブフィルター中はチャンネルが一覧から消える）
- 解除失敗時はダイアログ内またはトーストでエラーメッセージを表示する

## 8. 各種状態の表示

### 8.1 チャンネルリスト読み込み中

- グループヘッダーのスケルトンローダーを2件、チャンネル行のスケルトンローダーを5件分表示する

### 8.2 チャンネルが0件（アクティブフィルター時）

```
+------------------------------------------------------------------+
|                                                                  |
|   チャンネルがまだ登録されていません。                                 |
|   設定画面からチャンネルを同期してください。  [設定画面へ →]            |
|                                                                  |
+------------------------------------------------------------------+
```

### 8.3 解除済みチャンネルが0件（解除済みフィルター時）

```
+------------------------------------------------------------------+
|                                                                  |
|   解除済みのチャンネルはありません。                                  |
|                                                                  |
+------------------------------------------------------------------+
```

### 8.4 APIエラー時

- チャンネル一覧の取得失敗：画面にエラーメッセージを表示する（「チャンネルの取得に失敗しました。再読み込みしてください。」）
- カテゴリ変更・登録解除の失敗：トースト通知でエラーメッセージを表示する

## 9. カテゴリナビゲーション

チャンネルリストの横にカテゴリ名を縦に並べたナビゲーションを表示する。カテゴリ数・チャンネル数が増加した際に、目的のカテゴリへ素早くアクセスできるようにする。

### 9.1 表示条件

- カテゴリグループが2つ以上ある場合にのみ表示する
- チャンネルが0件の場合やカテゴリグループが1つの場合は非表示

### 9.2 PC表示（768px以上）

- チャンネルリストの左側に sticky サイドバーとして表示する
- カテゴリの表示順は既存の `sortOrder` 順に従う（未分類は末尾）
- カテゴリ名をクリックすると、該当カテゴリグループへスムーズスクロールする
- スクロール先のカテゴリが折りたたまれている場合は自動展開してからスクロールする

### 9.3 スマートフォン表示（768px未満）

- 右下にフローティングボタン（リストアイコン）を表示する
- フローティングボタンをタップすると左からドロワー（Sheet）でカテゴリ一覧を表示する
- ドロワー内のカテゴリを選択すると、ドロワーを閉じて該当グループへスクロールする
- 折りたたまれているカテゴリは自動展開してからスクロールする

## 11. データ取得・更新フロー

| 操作 | APIコール |
|---|---|
| 初回表示 | `GET /api/channels?isActive=true` ＋ `GET /api/categories`（グルーピング・セレクトボックス用） |
| フィルター変更 | `GET /api/channels?isActive={true\|false}` |
| カテゴリ変更 | `PATCH /api/channels/{channelId}` → リスト再取得 |
| 登録解除 | `PATCH /api/channels/{channelId}` (`isActive: false`) → リスト再取得 |

- チャンネル一覧とカテゴリ一覧は TanStack Query でキャッシュし、画面のフォーカス復帰時に再取得する
- カテゴリ変更は楽観的更新（セレクトボックスの値）を適用し、リスト再取得後に最終的な表示位置に更新する

## 12. 関連API

| メソッド | パス | 用途 |
|---|---|---|
| GET | `/api/channels` | チャンネル一覧取得（`isActive` フィルター） |
| GET | `/api/categories` | カテゴリ一覧取得（グルーピング・セレクトボックスの選択肢用） |
| PATCH | `/api/channels/{channelId}` | カテゴリ割り当て変更・登録解除 |
