# Google Cloud Console セットアップガイド

YouTube連携に必要なGoogle Cloud側の設定手順を説明する。

> **注**: Google Cloud Consoleの画面は随時更新されるため、本書の記載と実際の画面構成が異なる場合がある。

## 前提条件

- Googleアカウントを持っていること

## 手順1: プロジェクト作成

1. [Google Cloud Console](https://console.cloud.google.com/) にアクセスし、Googleアカウントでログインする
2. 画面上部のプロジェクト選択ドロップダウンをクリック
3. 「新しいプロジェクト」をクリック
4. プロジェクト名を入力（例: `MySubChs`）し、「作成」をクリック
5. 作成後、上部のドロップダウンから作成したプロジェクトに切り替える

## 手順2: YouTube Data API v3 の有効化

1. ナビゲーションメニュー（≡）→「APIとサービス」→「ライブラリ」を開く
2. 検索バーに「YouTube Data API v3」と入力し、検索結果から選択する
3. 「有効にする」をクリック

## 手順3: OAuth同意画面の設定

1. ナビゲーションメニュー（≡）→「APIとサービス」→「OAuth同意画面」を開く
2. 「開始」をクリック
3. アプリ情報を入力する
   - **アプリ名**: `MySubChs`
   - **ユーザーサポートメール**: 自分のメールアドレスを選択
4. 「対象」セクションで **「外部」** を選択する
5. デベロッパーの連絡先情報に自分のメールアドレスを入力する
6. 「データアクセス」に進み、「スコープを追加または削除」をクリック
   - フィルターに `youtube.readonly` と入力
   - `.../auth/youtube.readonly` にチェックを入れる
   - 「更新」をクリック
7. 「対象」画面に戻り、「Add users」からテストユーザーとして自分のGoogleメールアドレスを追加する

> **テストユーザーについて**: 「外部」で作成したアプリはデフォルトで「テストモード」となる。Googleの公開審査を通さなくても、テストユーザーに登録したアカウントのみアプリを利用できる。個人用アプリであれば審査は不要。

## 手順4: OAuthクライアントIDの作成

1. ナビゲーションメニュー（≡）→「APIとサービス」→「認証情報」を開く
2. 「+ 認証情報を作成」→「OAuthクライアントID」を選択
3. 以下を設定する
   - **アプリケーションの種類**: 「ウェブ アプリケーション」
   - **名前**: `MySubChs`（任意）
   - **承認済みのリダイレクトURI**: `http://localhost:3000/api/auth/callback/google`
     - 注: 「承認済みのJavaScript生成元」ではなく「承認済みのリダイレクトURI」に入力すること
4. 「作成」をクリック
5. 表示される **クライアントID** と **クライアントシークレット** を控える

## 手順5: .env ファイルへの設定

`.env` ファイルの以下の項目を設定する:

```env
# UIプレビューモードを無効化（行を削除またはコメントアウト）
# DEV_BYPASS_AUTH=true

# Google OAuth（手順4で取得した値を設定）
GOOGLE_CLIENT_ID=取得したクライアントID
GOOGLE_CLIENT_SECRET=取得したクライアントシークレット
```

`NEXTAUTH_SECRET` と `NEXTAUTH_URL` は [README.md](../../README.md) の共通手順で設定済みのため、変更不要。
