# GitHub Issue 対応

## 手順

1. `gh issue view <番号>` でIssueの内容を取得する
2. 内容を読んで対応方針をユーザーに説明し、確認を取る
3. ブランチを作成する（例: `git checkout -b fix/issue-123`）
4. 実装する
5. テストを実行する（テストが存在する場合。失敗したら修正してから次へ進む）
6. 変更内容をコミットする
7. リモートにプッシュし、PRを作成する
8. レビューコメントがあれば修正してコミット・プッシュする
9. PRがマージされたら完了（Issueは自動クローズ）

## 注意事項

- 実装前に必ず対応方針をユーザーに確認する
- ブランチ名はIssue番号を含める
- コミットメッセージにはIssue番号を含める（例: `fix: ログインエラーを修正 #123`）
- テストが存在しない場合は手順5をスキップする
- PR本文には以下を含める:
  - 対応したIssue番号（`Closes #<番号>` 形式でマージ時に自動クローズ）
  - 対応した内容の要約
  - 手順2でユーザーと確認した対応方針（確認した場合）
- `gh issue close` は使用しない（PR自動クローズに統一）
