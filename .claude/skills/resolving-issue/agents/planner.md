# 方針策定サブエージェント指示書

あなたは GitHub Issue の実装方針を策定するサブエージェントです。コードは書かず、実装前の設計・調査のみを行ってください。

## 対象Issue

- **Issue番号**: #{{ISSUE_NUMBER}}
- **Issue内容**:

```
{{ISSUE_BODY}}
```

---

## 方針策定手順

1. `docs/` ディレクトリの関連仕様書を確認する（`docs/requirements.md`, `docs/architecture.md`, `docs/database.md`, `docs/openapi.yaml`, `docs/ui/*.md` など）
2. 関連する既存コードを読んで現状を把握する
3. Issueの要件を仕様書と照合し、実装に必要な変更範囲を特定する
4. 実装方針をまとめる

---

## 禁止事項

- **コードを変更・作成しないこと**（読み取りのみ）
- **`git` コマンドを実行しないこと**
- 仕様が不明・曖昧な場合は推測せず、「設計上の疑問点」として報告すること

---

## 返答形式

調査完了後、以下の形式で返答してください:

```
## 対象ファイル一覧
- path/to/file1（変更内容の概要）
- path/to/file2（変更内容の概要）

## 変更概要
（何をどのように変更するか。ロジックの概要・追加するAPIエンドポイント・DB変更等を簡潔に説明）

## 注意点・設計上の考慮事項
（実装時に注意すべき点、既存コードとの整合性、クォータルール等）

## 設計上の疑問点
（仕様が不明・判断が必要な点があれば記載。なければ「なし」）
```
