# GitHub Issue 作成

## 手順

1. ユーザーから課題の概要を受け取る
2. 以下のテンプレートに沿ってIssue本文を生成する
3. 内容をユーザーに確認する
4. `gh issue create` コマンドで作成する

## Issueテンプレート

### 概要
（課題を1〜2文で整理）

### 背景・目的
（なぜこの課題が必要か）

### 詳細 / 再現手順
（具体的な内容）

### 完了条件
（どうなれば解決か）

### 影響範囲
（関連するファイル・機能）

## タイトルルール

タイトルには種別を示す接頭語を付ける。

| 接頭語 | 用途 |
|---|---|
| `[spec]` | 仕様に関する課題 |
| `[bug]` | バグ・不具合 |
| `[feat]` | 機能追加・改善 |
| `[chore]` | 設定・依存関係・リファクタなど |

例: `[spec] ポーリング間隔の仕様が未定義`

## ラベル運用ルール

Issue 作成時は、課題の種別に応じて以下のラベルを付与する。

| ラベル | 色 | 用途 |
|---|---|---|
| `spec` | 緑 `#0E8A16` | 仕様書に関する課題（矛盾・未定義・不足など詳細は本文に記載） |
| `bug` | — | コードのバグ・不具合 |
| `enhancement` | — | 機能追加・改善 |

その他の GitHub デフォルトラベル（`documentation`, `question` など）は必要に応じて使用する。

### 新しいラベルを作成する場合

- `gh label create "ラベル名" --description "説明" --color "HEXコード"` で作成する
- このセクションに追記して運用ルールを最新に保つ

## 実行コマンド例

```
gh issue create --title "[spec] タイトル" --label "spec" --body "本文"
```
